# 题目二实施方案：基于 LeRobot 的 ACT 策略跨环境泛化

> 目标平台：本地 Mac Air（写代码 / 写报告 / 看曲线）+ AutoDL（训练与评测）
> 数据集：CALVIN（环境 A/B/C/D）
> 算法：LeRobot 内置 ACT（Action Chunking Transformer）

---

## 0. 任务拆解与关键设计决策

题目要做三件事：

1. **基础模型**：只用 **环境 B** 的数据训练一个 ACT 策略。
2. **联合模型**：用 **环境 A+B+C** 混合数据，**相同网络结构与超参数** 再训一个。对比两者的训练收敛（重点 Action L1 Loss 曲线）。
3. **零样本测试**：两个模型都到 **未见过的环境 D** 上跑，比成功率 / 动作误差；分析 ACT 的 action chunking 在视觉分布偏移下的鲁棒性。

### 必须先定下来的三个简化决策（直接决定工作量大小）

**决策 1 — 评测方式：动作误差 vs 仿真成功率。**
CALVIN 原生评测要在 `calvin_env`（PyBullet）里跑 rollout 算成功率，服务器要配无头 EGL 渲染，调通成本高。题目原文是「成功率**或**动作误差」，二选一即可。

- 🟢 **推荐主路线（轻量）**：在环境 D 的 validation 集上做**离线动作误差评测**（Action L1 / MSE）。不需要仿真器，几分钟跑完，曲线干净，足以支撑「跨环境泛化」的全部分析。
- 🟡 可选加分项：调通 `calvin_env` 跑少量 rollout 出成功率，作为补充。建议主路线跑通、报告写完后再做。

**决策 2 — 是否用语言条件。**
CALVIN 是语言条件长程任务，ACT 原本是单任务短程动作分块，二者有错配。最省心且符合「轻量级 ACT」定位的做法：**把任务建模为纯视觉→动作的模仿学习**，忽略语言指令，state = 机器人本体感知（proprioception），action = 数据集里的 7 维 action（末端位姿增量 + 夹爪）。报告里需要说明这个建模选择及其局限。

**决策 3 — 数据规模。**
完整 `task_ABC_D` 约 500GB+，AutoDL 上既贵又慢。题目明确允许**降分辨率 + 子采样**。计划：先用 debug 数据集（~1.3GB）把全流程跑通，再对正式数据**按比例抽取 episode**（例如每个环境抽 N 条）来训练。

## 1. 阶段 0：AutoDL 环境配置

### 1.1 选机器
- GPU：单卡 RTX 3090 / 4090（24GB）足够，ACT 很轻。
- 镜像：选 PyTorch 2.x + CUDA 11.8/12.1 的基础镜像。
- 系统盘之外**务必挂载数据盘**，确认可用空间 ≥ 你打算下载的 CALVIN 子集大小（debug 调通后再决定正式规模）。

### 1.2 装 LeRobot
```bash
conda create -y -n lerobot python=3.10
conda activate lerobot

# 方式 A：源码安装（推荐，方便改数据转换/评测脚本）
git clone https://github.com/huggingface/lerobot.git
cd lerobot
pip install -e ".[act]"     # 安装 ACT 相关依赖

# 验证
lerobot-train --help        # 新版命令；若不存在则用 python -m lerobot.scripts.train --help
```
> ⚠️ LeRobot 接口更新很快。先 `pip show lerobot` 记下版本号，写进 README。命令到底是 `lerobot-train` 还是 `python lerobot/scripts/train.py`，以 `--help` 跑通的为准。

### 1.3 装实验记录工具（报告强制要求）
```bash
pip install swanlab
swanlab login            # 粘贴你的 API key
```

## 2. 阶段 1：数据准备（本任务最难、最花时间的部分）

### 2.1 下载 CALVIN 数据
```bash
git clone --recurse-submodules https://github.com/mees/calvin.git
cd calvin/dataset

# 先下 debug 调通流程（~1.3GB）
sh download_data.sh debug

# 正式数据：task_ABC_D 训练集含 A/B/C、验证集是 D —— 正好对应本任务
# sh download_data.sh ABC     # 约 500GB+，确认磁盘和预算后再下
```
数据结构（关键）：
```
task_ABC_D/
  training/      # 含 A、B、C 三个环境的 episode
    episode_xxxxxxx.npz
    scene_info.npy        # ★ 记录每段 episode 属于哪个场景(A/B/C)
    ...
  validation/    # 环境 D —— 作为 zero-shot 测试集
```

### 2.2 按环境拆分（题目核心：分出「B」和「A+B+C」）
- `scene_info.npy` 里有 `calvin_scene_B` / `calvin_scene_C` / `calvin_scene_D` 等键，给出每个环境对应的 episode 编号区间。
- 据此生成两份 episode 索引清单：
  - `split_B.txt`：只含环境 B 的 episode。
  - `split_ABC.txt`：A+B+C 全部 episode（可对每个环境等量抽样以控制规模）。

> 这一步要写一个小脚本读 `scene_info.npy` 输出索引清单（我可以帮你写）。

### 2.3 转成 LeRobotDataset 格式
LeRobot 不原生支持 CALVIN，需写一个转换脚本，把每条 episode 转为 LeRobotDataset：
- `observation.images.<cam>`：RGB 静态相机 / 手腕相机（**此处降分辨率**，如 200×200 → 96×96 省显存）。
- `observation.state`：机器人本体感知向量。
- `action`：CALVIN 的 7 维 action（相对末端位姿 6 维 + 夹爪 1 维）。
- 用 `LeRobotDataset.create(...)` 逐帧 `add_frame` + `save_episode`。
- 产出两个数据集：`calvin_B` 与 `calvin_ABC`（本地路径或推到自己的 HF Hub repo）。

> 转换脚本是整个任务的工程核心，我可以基于你的 LeRobot 版本帮你写好并在 debug 数据上验证。

## 3. 阶段 2：训练两个模型

两次训练**只有数据不同**，其余完全一致（这是对比的前提）。

```bash
# 模型 1：仅环境 B
lerobot-train \
  --dataset.repo_id=local/calvin_B \
  --policy.type=act \
  --output_dir=outputs/act_B \
  --job_name=act_calvin_B \
  --policy.device=cuda \
  --batch_size=8 \
  --steps=100000

# 模型 2：环境 A+B+C（其余参数与上面逐字相同）
lerobot-train \
  --dataset.repo_id=local/calvin_ABC \
  --policy.type=act \
  --output_dir=outputs/act_ABC \
  --job_name=act_calvin_ABC \
  --policy.device=cuda \
  --batch_size=8 \
  --steps=100000
```
> 注：实际用本仓库的 `src/train_base.py`（已集成 SwanLab、语言条件、最优权重保存）。
要点：
- 把 `batch_size`、`steps`、`chunk_size`、`lr`、optimizer 等记进报告的超参数表。
- 两个 run 都进同一个 SwanLab project，方便叠加画 Action L1 Loss 对比曲线。
- 先用 debug 数据 + `--steps=2000` 验证能否正常出 loss，再上正式数据。

## 4. 阶段 3：环境 D 上的 Zero-shot 评测

### 主路线（离线动作误差，推荐）
写一个评测脚本：加载 checkpoint → 遍历环境 D 的 validation episode → 模型预测 action chunk → 与真值 action 比 **L1 / MSE**。
- 对 `act_B` 和 `act_ABC` 各跑一遍，得到两组误差。
- 输出：整体 L1、按 chunk 内时间步分解的 L1（用来讲 action chunking 的鲁棒性）。

### 加分路线（仿真成功率，可选）
```bash
# 需先配好 calvin_env 的无头渲染
export EGL_DEVICE_ID=0
# 用 CALVIN 的 evaluation 协议在环境 D 跑 rollout，统计 task success rate
```

### 要重点分析的现象
- 联合模型（A+B+C）在 D 上的误差应明显低于仅 B 的模型 → 说明多环境数据缓解视觉分布偏移。
- 结合 action chunking 机制讨论：一次预测一段动作、对单帧视觉噪声/偏移不那么敏感，为何相对鲁棒。

## 7. 踩坑清单（提前知道能省很多时间）

1. **LeRobot 版本漂移**：命令名、配置字段经常变，一切以你装的版本 `--help` 为准；版本号写进 README。
2. **CALVIN 数据巨大**：别一上来下 ABCD（~660GB）。先 debug，再按需抽样 ABC。
3. **calvin_env 无头渲染**：服务器没显示器，要配 EGL；这也是为什么主路线建议走离线动作误差、绕开仿真。
4. **「相同超参数」是对比的命门**：两次训练除数据集外其他参数必须逐字一致，否则对比不成立。
5. **action 维度对齐**：CALVIN 的相对/绝对 action 与夹爪要搞清楚，转换时别弄错归一化，否则 loss 看似收敛但预测全错。
6. **降分辨率**：转换阶段就把图缩小（如 96×96），训练显存和速度都受益，报告里说明即可。
